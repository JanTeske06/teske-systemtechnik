eBay Automation Pipeline

Automated listing pipeline for an eBay seller managing 200+ products daily — 95% time saved.

Client

Private client via Upwork

Duration

3 weeks

Budget

$2,750

Technology

  • Python
  • Selenium
  • FastAPI
  • PostgreSQL

Date

2024-08-15

The challenge

The client runs a mid-sized eBay shop with over 2,000 active listings and used to maintain every product manually: adjusting prices, updating stock, posting new items. The daily effort: about 6 hours. That didn’t just cost time — it produced errors: incorrect prices, forgotten updates, accidentally deleted listings.

The solution

I built a Python-based automation pipeline that covers the complete lifecycle of a listing:

  1. CSV import from the client’s ERP system
  2. Validation of every row against eBay’s category requirements
  3. Selenium-based interaction with the eBay seller portal (because the official API was too limited)
  4. PostgreSQL database as a single source of truth for listing state
  5. FastAPI dashboard so the client can see updates in real time

The entire process now runs automatically at night via a cron job — the client just glances at the dashboard in the morning.

The result

  • 95% time saved (from 6h down to ~20 minutes of review per day)
  • Error rate dropped from ~8% to under 1%
  • New listings go live within 2 hours of CSV upload
  • ROI for the client reached after 6 weeks

Technical details

The architecture was intentionally simple — a monolithic Python service on a small VPS hosting both the Selenium workers and the FastAPI dashboard. No Kubernetes, no message broker, no microservices — because the problem didn’t need them.

The biggest technical hurdle was eBay’s rate limiting and their dynamic DOM, which occasionally changed between deployments. I built a selector wrapper that automatically notifies on failure and tries a fallback path.
